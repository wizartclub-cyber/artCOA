This guide is for the gallery operator who manages signing keys, the certificate registry and settings. The admin pages are password-protected.

1 · Signing keys — one per artist

  • Each artist (or gallery) has their own ECDSA P-256 key pair. They generate it in keys.html, keep the private key, and send you their public key + their ID (the initials used in the Edition ID, e.g. sd).
  • In admin-keys.html → Approve a new signer: pick Type, Name, ID, paste the public key → Validate & approve.
  • The trusted list lives in artist-keys.js. After approving, click Download artist-keys.js and deploy it (commit/push). Only the deployed file is trusted — nothing is trusted from the QR link.
  • The generator shows “✓ Signed (verifies)” only if the pasted private key matches that artist’s registered public key; otherwise “mismatch” or “artist not registered”.

2 · Registry of issued certificates

Every signed certificate is recorded to the Registry tab of your Google Sheet (id, edition, artwork data, image URL, verification link). This is your master list of everything issued. Unsigned/test certs are not recorded.

3 · Revoking a certificate

  • In the Registry tab, put anything (e.g. a date) in the Revoked column next to the certificate.
  • The verification page checks the registry and shows “Revoked” for that certificate — even if the signature is still valid.
  • Requires the backend (Apps Script) to be redeployed with the current code.

4 · Backend (Google Apps Script)

A free Apps Script web app stores the anonymous analytics and the registry in one Google Sheet. Setup instructions are inside backend/apps-script.gs. After any change to the backend code, redeploy: Deploy → Manage deployments → ✏️ → Version: New version → Deploy. The URL stays the same.

5 · Usage analytics (admin.html)

Anonymous events only — category, status, language, device. No names, no IP by default, and only after the buyer accepts the consent banner. View/filter and export CSV; the full data is in the Google Sheet.

6 · Artwork images (Cloudinary)

Images are resized for print and uploaded to Cloudinary; the URL is embedded in the (signed) certificate so the verification page shows the photo. Config in index.html: CLOUDINARY_CLOUD + CLOUDINARY_PRESET (an unsigned upload preset).

7 · Admin password

admin.html and admin-keys.html are gated by a passphrase (admin-gate.js), stored only as a SHA-256 hash. Change it: on the gate click Set / change passphrase → copy the new hash into admin-gate.js → deploy. The default is change-me-now — change it.

Honest scope. The password is a client-side deterrent, not server-grade security (the site is static). The real control over what is trusted is GitHub repo (deploy) access — only you can publish artist-keys.js. For true page-level protection use Cloudflare Access.

8 · Security model (summary)

  • Per-artist asymmetric signatures (ECDSA P-256). Forgery is infeasible without the artist’s private key; any change to the data invalidates the signature; an unregistered artist cannot verify.
  • The trust anchor is the committed artist-keys.js (controlled by repo access), never the QR.
  • Limitations: certificate data still travels in the QR link; the registry & revocation need the backend online.

Цей посібник — для оператора галереї, який керує ключами підпису, реєстром сертифікатів і налаштуваннями. Адмін-сторінки захищені паролем.

1 · Ключі підпису — у кожного художника свій

  • У кожного художника (чи галереї) — власна пара ECDSA P-256. Він генерує її в keys.html, приватний ключ лишає собі, а тобі надсилає публічний ключ + свій ID (ініціали з Edition ID, напр. sd).
  • У admin-keys.html → Approve a new signer: обери тип, ім'я, ID, встав публічний ключ → Validate & approve.
  • Довірений список — у файлі artist-keys.js. Після апруву натисни Download artist-keys.js і задеплой (commit/push). Довіряється лише задеплоєний файл — з QR ключі не беруться.
  • Генератор показує «✓ Signed (verifies)» лише якщо приватний ключ відповідає зареєстрованому ключу цього художника; інакше «mismatch» або «художника не зареєстровано».

2 · Реєстр виданих сертифікатів

Кожен підписаний сертифікат записується у вкладку Registry твоєї Google-таблиці (ID, тираж, дані твору, URL фото, посилання для перевірки). Це твій повний список усього виданого. Непідписані/тестові не записуються.

3 · Відкликання сертифіката

  • У вкладці Registry познач будь-що (напр. дату) у колонці Revoked навпроти сертифіката.
  • Сторінка перевірки звіряє реєстр і показує «Відкликано» для цього сертифіката — навіть якщо підпис ще дійсний.
  • Потрібно, щоб бекенд (Apps Script) був перерозгорнутий із поточним кодом.

4 · Бекенд (Google Apps Script)

Безкоштовний веб-застосунок Apps Script зберігає знеособлену аналітику і реєстр в одній Google-таблиці. Інструкція — всередині backend/apps-script.gs. Після будь-якої зміни коду бекенда — перерозгорни: Deploy → Manage deployments → ✏️ → Version: New version → Deploy. URL лишається той самий.

5 · Аналітика використання (admin.html)

Лише знеособлені події — категорія, статус, мова, девайс. Без імен та IP за замовчуванням і лише після згоди покупця в банері. Перегляд/фільтр + експорт CSV; повні дані — у Google-таблиці.

6 · Зображення творів (Cloudinary)

Фото зменшується для друку й завантажується в Cloudinary; URL вбудовується в (підписаний) сертифікат, щоб сторінка перевірки показувала твір. Налаштування в index.html: CLOUDINARY_CLOUD + CLOUDINARY_PRESET (unsigned пресет).

7 · Пароль адмінки

admin.html і admin-keys.html закриті паролем (admin-gate.js), що зберігається лише як SHA-256-хеш. Зміна: на гейті Set / change passphrase → встав новий хеш у admin-gate.js → деплой. Дефолт — change-me-now, зміни його.

Чесно про межі. Пароль — клієнтський стримувач, а не серверний захист (сайт статичний). Справжній контроль над тим, що довіряється, — це доступ до GitHub-репо (тільки ти публікуєш artist-keys.js). Для повноцінного захисту сторінок — Cloudflare Access.

8 · Модель безпеки (стисло)

  • Асиметричні підписи на кожного художника (ECDSA P-256). Підробка без приватного ключа неможлива; зміна даних робить підпис недійсним; незареєстрований художник не пройде перевірку.
  • Якір довіри — задеплоєний artist-keys.js (керується доступом до репо), ніколи не QR.
  • Обмеження: дані сертифіката досі в QR-посиланні; реєстр і відкликання потребують онлайн-бекенда.
