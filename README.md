# ArtCOA — Certificate of Authenticity

Статичний сайт для створення **двосторінкових сертифікатів автентичності (CoA)** з QR-перевіркою.
Працює повністю в браузері, **без сервера й бази даних**. Хостинг — GitHub Pages (безкоштовно).
Розроблено для **Smalta Art Gallery**.

**Живі адреси:**
- 🌐 Генератор: https://wizartclub-cyber.github.io/artCOA/
- ✅ Перевірка (через QR): https://wizartclub-cyber.github.io/artCOA/verify.html
- 📖 Довідка: https://wizartclub-cyber.github.io/artCOA/help.html
- 📊 Панель статистики: https://wizartclub-cyber.github.io/artCOA/admin.html

## Що робить програма

- **Сторінка 1 — Certificate of Authenticity:** усі дані твору, категорія, статус унікальності, ідентифікатори, QR-код, лінії для підписів.
- **Сторінка 2 — Artwork Identification:** зображення твору, ключові ідентифікатори та підписи.
- **PDF** генерується друком браузера → **чіткий векторний текст** (виділяється), рівно 2 сторінки A4.
- **QR-код** веде на `verify.html`; дані сертифіката «зашиті» в посилання.

### Мови
English / Українська / Slovenčina.
- **Форма** показується обраною мовою.
- На **сертифікаті** підписи: англійською для EN; двомовні (`обрана / English`) для UK і SK.

### Категорія мистецтва (Category)
Кожен твір відноситься до однієї з 8 міжнародних категорій артринку. На сертифікаті
друкується обрана категорія у форматі `КОД — Назва` (напр. `MIX — Mixed Media & Phygital Art`).

| Код | Категорія |
|---|---|
| PTG | Paintings / Живопис |
| DRW | Drawings & Works on Paper / Графіка |
| MIX | Mixed Media & Phygital Art / Змішана техніка та Фіджитал-арт |
| PRN | Prints & Multiples / Тиражна графіка |
| SCL | Sculptures & 3D Objects / Скульптура |
| PHO | Photography / Фотографія |
| DIG | Digital Art / Цифрове мистецтво |
| DEC | Design & Decorative Art / Дизайн та Декор |

При виборі категорії поле «Матеріали та техніка» показує приклад-підказку
(напр. для MIX: *Giclée on canvas, hand-embellished with acrylic and gold leaf*).

### Статус унікальності (Edition Status)
На сертифікаті явним рядком: `Unique piece (1/1)` або `Limited Edition — copy 15 of 50`.
Це головний ціноутворюючий фактор для колекціонера.

### Edition ID (формується автоматично)
`ініціали-рік-№твору-код-розмірСм-тираж` → напр. `sd-2026-001-EDH-50x70-15`.
**Номер сертифіката** = Edition ID + номер копії → `…-15-1`.

Код видання підставляється за статусом:

| Status edition | Код |
|---|---|
| Limited Edition | ED |
| Hand-embellished Limited Edition | EDH |
| Open Edition | OED |
| Original | OR |
| Artist Proof | AP |
| Digital | DIG |
| Reproduction | REP |

## Як користуватись

1. Обери мову (вгорі зліва).
2. Заповни дані твору. Обери **категорію** — поле Medium підкаже приклад. Edition ID і № сертифіката рахуються самі.
3. На стор. 2 додай зображення твору («Вибрати файл»).
4. Дата підставляється поточна (за часовим поясом користувача, формат `dd.mm.yyyy`).
5. Встав **приватний ключ підпису** (згенеруй одноразово в `keys.html`) — тримай у таємниці. Можна поставити «Запам'ятати на пристрої».
6. **Save as PDF** → у діалозі друку обери «Зберегти як PDF».
   *Порада (одноразово): More settings → Margins: None, вимкни Headers and footers.*
   **Send PDF** — поділитися готовим файлом. **Copy verify link** — скопіювати посилання.
7. Покупець сканує QR → бачить сторінку перевірки. Щоб довести справжність, вводить твій ключ → «Автентичний».

## Аналітика використання

Сайт статичний, тож сам по собі не веде спільну таблицю. Збір улаштовано так:

- `analytics.js` у браузері збирає **знеособлену подію** (категорія, мова, тип девайсу),
  зберігає локально (`localStorage`) і надсилає на бекенд. **Лише після згоди користувача.**
- `admin.html` — панель перегляду (локальні події) + експорт CSV.
- `backend/apps-script.gs` — Google-таблиця = центральна таблиця всіх користувачів.

> **Збір імен покупців за замовчуванням ВИМКНЕНО** (рішення щодо GDPR — див. нижче).

### Підключити центральну таблицю
1. Виконай кроки всередині `backend/apps-script.gs` → отримаєш «Web app URL».
2. У `analytics.js` встав його у `ANALYTICS_ENDPOINT` (вже підключено для цього проекту).
3. Після зміни коду бекенда **перерозгорни**: Deploy → Manage deployments → ✏️ → New version.

## Приватність (GDPR)

- На сайті є **банер згоди**: збір даних відбувається **лише після «Accept»**.
- Натискання **«Decline»** → нічого не збирається, не зберігається й не надсилається; сайт працює повноцінно.
- Рішення можна змінити посиланням **«Privacy & data»**.
- **На етапі валідації імена покупців та IP НЕ збираються.** Збирається лише знеособлена
  статистика (категорія, мова, девайс), яка не дозволяє ідентифікувати особу.
- Якщо в майбутньому збиратимуться персональні дані (імена, IP) — обов'язково:
  додати у Privacy Policy ЩО / НАВІЩО / СКІЛЬКИ зберігається, та правову підставу обробки.

## Безпека та обмеження

- Сертифікати підписуються **асиметрично — ECDSA P-256** (WebCrypto). Галерея підписує **приватним ключем**, а `verify.html` перевіряє **публічним ключем автоматично** — покупцю не потрібен секрет, підробка без приватного ключа неможлива, зміна даних робить підпис недійсним.
- Ключі: `keys.html` генерує пару; **приватний** ключ ніколи не комітиться (локально в `gallery-private-key.txt`, у `.gitignore`), **публічний** — у `verify.html` (`GALLERY_PUBKEY`).
- Обмеження: дані досі в QR-посиланні; немає центрального реєстру/відкликання (буде в серверній версії).
- Сторінка перевірки не показує зображення твору (його немає в QR — лише текст).

## Що далі (серверна версія)
- Реєстр сертифікатів на PostgreSQL (Hetzner).
- QR веде на короткий ID, дані (включно із зображенням) — з бази.
- Відкликання сертифікатів, акаунти галерей, оплата.
