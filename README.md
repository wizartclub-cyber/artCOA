# ArtCert MVP — Certificate of Authenticity

Статичний сайт для створення сертифікатів автентичності (CoA) з QR-перевіркою.
Працює повністю в браузері, **без сервера й бази даних**. Хостинг — GitHub Pages (безкоштовно).

## Що це робить

1. `index.html` — генератор: заповнюєш форму → бачиш live-прев'ю → друкуєш у PDF.
2. `verify.html` — сторінка перевірки: відкривається через QR-код, читає дані сертифіката з посилання.

Мови: English + Українська. Дизайн: нейтральний музейний.

---

## Як опублікувати (без терміналу, ~10 хвилин)

### Крок 1. Створи репозиторій
1. Зайди на https://github.com → кнопка **New** (зелена).
2. Repository name: `artcert`
3. Постав **Public**.
4. **Create repository**.

### Крок 2. Завантаж файли
1. На сторінці репозиторію натисни **uploading an existing file**.
2. Перетягни `index.html` і `verify.html`.
3. Внизу натисни **Commit changes**.

### Крок 3. Увімкни GitHub Pages
1. Вкладка **Settings** (вгорі репозиторію).
2. Зліва — **Pages**.
3. Source: **Deploy from a branch**.
4. Branch: **main** → папка **/ (root)** → **Save**.
5. Зачекай ~1 хвилину. З'явиться адреса:
   `https://ТВІЙ-ЛОГІН.github.io/artcert/`

### Крок 4. Впиши свою адресу перевірки
1. Відкрий свій сайт `…/artcert/`.
2. У полі **Verification page base URL** встав:
   `https://ТВІЙ-ЛОГІН.github.io/artcert/verify.html`
3. Тепер QR-коди вестимуть на твою сторінку перевірки.

## Як користуватись

1. Заповни форму (художник, назва, рік тощо).
2. Впиши **секретний ключ підпису** — придумай свій, тримай у таємниці.
   Один і той самий ключ підтверджує всі твої сертифікати.
3. **Save as PDF** → у діалозі друку обери «Зберегти як PDF».
4. Покупець сканує QR → бачить сторінку перевірки.
5. Щоб довести справжність — даєш покупцю свій ключ, він вводить його на сторінці
   перевірки. Якщо підпис співпадає → «Автентичний».

## Чесні обмеження цього MVP

- Це **валідація попиту**, а не криптографічний захист. Дані «зашиті» в QR-посиланні.
- Будь-хто технічно може згенерувати схожий QR — захист дає лише секретний ключ підпису.
- Не можна «відкликати» сертифікат централізовано (немає сервера).
- Коли художники почнуть платити → додаємо сервер + базу (твій Hetzner стек).

## Що далі (коли підтвердиться попит)
- Реєстр сертифікатів на PostgreSQL (Hetzner).
- Реальна верифікація: QR веде на короткий ID, дані з бази.
- Можливість відкликання, акаунти галерей, Stripe.
