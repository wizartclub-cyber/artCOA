ArtCOA — Signing keys

← Back to generator

Certificates are signed with an ECDSA P-256 key pair. The private key signs (keep it secret); the public key is what verify.html uses to check signatures (safe to share).

1 · Generate a new key pair

Do this once. It runs entirely in your browser — keys never leave this page.

2 · Private signing key — keep secret

Never share or commit this. Paste it into the generator’s “Private signing key” field to sign certificates. Anyone with this key can issue valid certificates.

3 · Public key — for verify.html

Safe to share. Send this to your developer to put into verify.html (the GALLERY_PUBKEY constant), so scanned certificates verify automatically.

Recover public key from a private key

Lost your public key but still have the private one? Paste the private key to derive the public key again.

Algorithm: ECDSA P-256, SHA-256. Keys are stored as base64url-encoded JWK.
